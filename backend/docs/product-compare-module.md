# DigiCompass 产品对比模块记录

记录日期：2026-07-11

## 1. 模块目标

产品对比模块用于辅助用户在多个数码产品之间做横向比较。当前版本支持从产品列表页和产品详情页加入对比，并在对比页展示产品基础信息、价格、核心参数和选购判断。

模块当前不新增后端接口。前端复用已有接口：

- `GET /api/products`
- `GET /api/products/{id}`

对比选择状态保存在浏览器 `localStorage`，不依赖用户登录状态。

## 3. 状态设计

`compareStore` 使用 `digicompass_compare` 作为本地存储键。

状态与方法：

| 名称 | 说明 |
| --- | --- |
| `selectedIds` | 已加入对比的产品 ID |
| `count` | 当前对比数量 |
| `canAdd` | 是否还能继续加入产品 |
| `isSelected(id)` | 判断产品是否已加入 |
| `addProduct(id)` | 加入产品，最多 3 个 |
| `removeProduct(id)` | 移除产品 |
| `clear()` | 清空对比 |

读取本地数据时会过滤非法 ID、去重，并限制最多 3 个。

## 4. 页面流程

用户端产品列表页的卡片上有“+ 对比”按钮。按钮点击后会阻止 `RouterLink` 默认跳转，只修改对比状态。

产品详情页提供“加入对比”按钮。产品加入后，按钮显示“已加入对比”，并显示“查看对比 x/3”入口。

对比页打开后，根据 `selectedIds` 调用 `GET /api/products/{id}` 获取产品详情。页面展示：

- 产品图片、名称、品牌和分类
- 推荐分
- 官方价
- 二手参考价
- 核心参数
- 选购判断标签

对比页支持移除单个产品、清空对比和返回产品库。

## 5. 差异展示

对比页会对每一行进行差异判断。判断方式为将同一行中各产品的展示值标准化为字符串，再比较去重后的数量。

当同一行存在不同值时：

- 行左侧标签使用浅绿色背景。
- 数据单元格使用浅色背景。
- “只看差异”开启后，只保留差异行。

产品头部行始终显示，用于确认当前正在对比的产品。
